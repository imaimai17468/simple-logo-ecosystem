# フィードバック依頼テンプレート

プロトタイプを共有する際に使えるテンプレート集。

---

## Slack / Discord 用

### シンプル版

```markdown
プロトタイプ触ってみてください！

URL: https://xxxx.trycloudflare.com

**見てほしいところ**
- [機能名や画面]

気づいたことあれば何でも教えてください！
```

### 詳細版

```markdown
## プロトタイプレビューのお願い

**URL**: https://xxxx.trycloudflare.com

### これは何？
[アプリの概要を1-2文で]

### 見てほしいポイント
- [ ] [具体的な機能1]
- [ ] [具体的な機能2]
- [ ] [具体的な画面]

### 教えてほしいこと
- 使いにくかったところ
- 「こうなってほしい」と思ったこと
- 良かったところ

### 注意事項
- 開発中なのでデータは保存されません
- 動かないところがあったら教えてください
- 有効期限: 今日中

よろしくお願いします！
```

## 口頭でフィードバックをもらう際のガイド

### 観察ポイント

```markdown
触ってもらいながら観察:
- どこで手が止まるか
- 何を探しているか
- 表情の変化
- 独り言（「あれ？」「えーと...」）
```

### 質問例

```markdown
**最初に**
「まず自由に触ってみてください。思ったことは何でも言ってください」

**途中で**
「今何をしようとしていますか？」
「何を探していますか？」

**終わった後**
「一番良かったところは？」
「一番困ったところは？」
「次にやりたいことは何でしたか？」
```

## Tips

### 効果的なフィードバックをもらうコツ

1. **具体的な質問をする**
   - NG: 「どうですか？」
   - OK: 「この画面で何をすればいいか分かりましたか？」

2. **タスクを与える**
   - 「写真を1枚講評してみてください」
   - 「履歴から過去の講評を見てみてください」

3. **最初は黙って見守る**
   - 説明したい気持ちを抑える
   - 困っているところを観察

4. **批判を歓迎する姿勢を見せる**
   - 「厳しい意見が一番助かります」
   - 「動かなかったところも教えてください」
